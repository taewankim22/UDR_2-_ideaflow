공통 계약 (FROZEN) — 양쪽 AI에 그대로 입력

이 두 블록이 백·프론트의 유일한 진실(source of truth)입니다. 바꾸면 양쪽 다 바꿔야 함.

// shared/types.ts — 백 · 프론트가 함께 import (호환성의 핵심)

type ApiResponse<T> =
  | { success: true;  data: T; meta?: { nextCursor?: string } }
  | { success: false; error: { code: ErrorCode; message: string } };

type ErrorCode = 'VALIDATION_ERROR' | 'UNAUTHORIZED' | 'FORBIDDEN'
  | 'INSUFFICIENT_POINTS' | 'NOT_FOUND' | 'CONFLICT' | 'AI_UNAVAILABLE';

type Category = 'AI'|'EDU'|'TRAVEL'|'ENV'|'HEALTH'|'ETC';
type Provider = 'email'|'kakao'|'google';

interface User { id:string; email:string; username:string;
  profileImageUrl:string|null; pointsBalance:number; }

interface IdeaCard {            // 피드 카드
  id:string; title:string; oneLine:string; category:Category;
  coverImageUrl:string|null; authorName:string;
  likeCount:number; commentCount:number;
  unlockCost:number; isUnlocked:boolean; createdAt:string; } // ISO8601 UTC

interface IdeaDetail extends IdeaCard {
  problem:string; solution:string; }  // 잠금 시 빈 문자열로 마스킹

interface Whiteboard { ideaId:string; updatedAt:string;
  nodes: { key:string; label:string; content:string }[]; } // 고정 7노드

interface AIEvaluation { ideaId:string; overallScore:number; // 0-100
  scores:{ market:number; feasibility:number; profitability:number;
           differentiation:number; growth:number };
  summary:string; suggestions:string[]; createdAt:string; }
// 포인트 정책 (DB 시드값 · 변동 가능) + 에러 HTTP 매핑
SIGNUP_BONUS +30   IDEA_CREATE +10   COMMENT_WRITE +3   LIKE_RECEIVED +1(상한10)
IDEA_UNLOCK -3      AI_EVALUATE -5    AI_COMPETITOR -10(P2)   AI_BUSINESS -15(P2)

400 VALIDATION_ERROR · 401 UNAUTHORIZED · 403 FORBIDDEN
402 INSUFFICIENT_POINTS · 404 NOT_FOUND · 409 CONFLICT · 503 AI_UNAVAILABLE
모든 응답은 위 ApiResponse 봉투로 통일 (raw 객체 그대로 반환 금지)

04

6/6 — 통합 & 호환성 테스트

합친 뒤 이 시나리오 하나가 끝까지 돌면 MVP 성공. 옆의 "호환성 함정"부터 점검하세요.

E2E 시나리오 (끝까지 1회)
  1. 가입 → 보너스 포인트(+30) 헤더에 보임
  2. 아이디어 작성 → +10P, 피드 최신 탭에 카드 노출
  3. 화이트보드 7노드 채우고 저장 → 새로고침해도 유지
  4. AI 평가 실행 → −5P 차감 → 종합점수·5항목·요약 표시
  5. 다른 글 열람 → −3P, 본문(problem/solution) 보임
  6. 좋아요·댓글 → 카운트 증가 + 작성자 적립
호환성 함정 — 합칠 때 여기서 깨진다
  • 필드명 양쪽 다 camelCase + 같은 types.ts. one_line↔oneLine 섞이면 즉시 깨짐
  • 날짜 전부 ISO8601 UTC 문자열로 주고받고, 화면 표시만 KST 변환
  • enum 값 문자열 정확히 일치 — category·provider·ErrorCode 오타 주의
  • 응답 봉투 모든 엔드포인트가 {success,data} 통일. 하나라도 raw 반환하면 프론트 파싱 터짐
  • CORS BE가 프론트 origin + Authorization 헤더 허용했는지 (통합 첫 에러 1순위)
  • AI 지연 Gemini 수 초 → 프론트 로딩 유지, BE 실패 시 환불+503 → 프론트 재시도
  • 포인트 중복 빠른 더블클릭(평가·언락 2회) → 버튼 disable + BE 멱등/원자 처리
  • .env 키(Gemini·JWT·DB) 누락 → 합치는 PC에서 한 번 더 확인
